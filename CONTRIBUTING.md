## üìã Guidelines

### Recipe Submissions

When submitting recipes, please include:

- **Prep and cook time**
- **Serving size**
- **Clear ingredient list** with measurements
- **Step-by-step instructions**
- **Nutritional highlights** (e.g., "High in fiber", "Good source of protein")
- **Budget estimate** if possible
- **Kid-friendliness rating** (tested with children)

### Content Standards

All contributions should:

- **Be accessible:** Use clear, simple language
- **Be practical:** Focus on real-world, achievable solutions
- **Be affordable:** Emphasize budget-friendly options
- **Be tested:** Share what has actually worked
- **Be inclusive:** Consider various dietary needs and restrictions

## üöÄ How to Submit Contributions

### Method 1: GitHub Issues (Easiest)

1. Go to the [Issues tab](https://github.com/kendychae/sustain-human-life/issues)
2. Click "New Issue"
3. Choose the appropriate template or create a custom issue
4. Describe your contribution clearly
5. Submit!

### Method 2: Pull Requests (For Code/Content Changes)

1. **Fork the repository**

   ```bash
   # Click the "Fork" button at the top right of the repository page
   ```

2. **Clone your fork**

   ```bash
   git clone https://github.com/YOUR-USERNAME/sustain-human-life.git
   cd sustain-human-life
   ```

3. **Create a new branch**

   ```bash
   git checkout -b feature/your-contribution-name
   ```

4. **Make your changes**

   - Edit files using your preferred text editor
   - Test your changes locally (open index.html in a browser)
   - Ensure formatting is consistent

5. **Commit your changes**

   ```bash
   git add .
   git commit -m "Add: Brief description of your changes"
   ```

6. **Push to your fork**

   ```bash
   git push origin feature/your-contribution-name
   ```

7. **Create a Pull Request**
   - Go to your fork on GitHub
   - Click "New Pull Request"
   - Provide a clear description of your changes
   - Submit!

### Method 3: Email Suggestions

If you're not comfortable with GitHub, you can suggest contributions by creating an issue describing what you'd like to share.

## ‚úÖ Pull Request Checklist

Before submitting a pull request, make sure:

- [ ] Content is original or properly credited
- [ ] Recipes have been tested with real families
- [ ] Text is clear, concise, and free of spelling errors
- [ ] Links work and open correctly
- [ ] Changes are consistent with existing site style
- [ ] Any new images are optimized for web (< 500KB)
- [ ] Accessibility standards are maintained (alt text, semantic HTML)

## üìù Code Style Guidelines

### HTML

- Use semantic HTML5 elements
- Include proper ARIA labels for accessibility
- Maintain consistent indentation (2 or 4 spaces)
- Add descriptive comments for complex sections

### CSS

- Follow existing naming conventions
- Group related styles together
- Use comments to separate major sections
- Ensure responsive design (mobile-first approach)
- Test on multiple screen sizes

### JavaScript

- Write clean, commented code
- Maintain compatibility with modern browsers
- Test functionality across devices
- Keep file sizes small for performance

## üé® Design Principles

- **Simplicity:** Clean, uncluttered layouts
- **Accessibility:** Readable fonts, good contrast, keyboard navigation
- **Mobile-First:** Design works on phones, tablets, and desktops
- **Performance:** Fast loading times, optimized images
- **Printability:** Meal plans and recipes should print well

## üß™ Testing

Before submitting, please test:

1. **Visual appearance** in multiple browsers (Chrome, Firefox, Safari, Edge)
2. **Mobile responsiveness** on phones and tablets
3. **Accessibility** using keyboard navigation
4. **Print layout** for meal plans and recipes
5. **Link functionality** - ensure all links work
